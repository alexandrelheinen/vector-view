# Contributing

Thank you for contributing to Vector View.

## Before you open a PR

- Build and run relevant tests (`cmake --build build`, `ctest` in `build/`).
- If you change demo behavior, verify it in Gazebo — do not rely on edited screenshots alone.
- Read **[docs/simulation-honesty.md](docs/simulation-honesty.md)**.

## Simulation honesty (required)

This project must not ship **fake** demo assets.

- **Solve** problems in the simulation, plugin, or capture pipeline — or **state clearly** what cannot be done yet.
- Do **not** post-process images/video to add force arrows, contact hints, or other visuals that the sim did not produce.
- Do **not** commit `docs/images/` comparison or demo frames unless they were generated by the documented capture scripts from a working sim run.

Agents working in Cursor must follow `.cursor/rules/simulation-honesty.mdc`.

## Layout and style

- Match existing naming and structure under `include/vectorview/`, `src/`, `assets/`, `scripts/`.
- Prefer minimal, focused diffs.
- C++17, CMake 3.16+.

## Commits

Use clear messages: what changed and why. Reference issues when applicable.
